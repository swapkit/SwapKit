# Migrating from SwapKit v3 to v4

This guide helps you upgrade your SwapKit integration from v3 to v4.

## Breaking Changes

### 1. Package Structure

v4 consolidates multiple packages into monolithic packages with submodule exports:

```diff
// v3
- import { NearPlugin } from "@swapkit/plugin-near";
- import { getToolboxByChain } from "@swapkit/toolbox-cosmos";
- import { keystoreWallet } from "@swapkit/wallet-keystore";

// v4
+ import { NearPlugin } from "@swapkit/plugins/near";
+ import { getCosmosToolbox } from "@swapkit/toolboxes/cosmos";
+ import { keystoreWallet } from "@swapkit/wallets/keystore";
```

### 2. Import Path Changes

All imports must be updated to use the new submodule syntax:

| v3 Package | v4 Import |
|------------|-----------|
| `@swapkit/plugin-*` | `@swapkit/plugins/*` |
| `@swapkit/toolbox-*` | `@swapkit/toolboxes/*` |
| `@swapkit/wallet-*` | `@swapkit/wallets/*` |

### 3. Plugin Interface Changes

The plugin initialization has changed:

```diff
// v3
- function plugin({ getWallet, config }: SwapKitPluginParams<ConnectConfig>) {
-   return { swap, supportedSwapkitProviders: [ProviderName.NEAR] };
- }

// v4
+ export function NearPlugin({ getWallet }: { getWallet: (chain: any) => any }) {
+   return { swap, supportedSwapkitProviders: [ProviderName.NEAR] };
+ }
```

### 4. Missing Features

Some v3 features are not yet available in v4:

- **NEAR Support**: Plugin created but requires API endpoint implementation
- **EVM API Balance**: Different architecture in v4, use standard balance methods

## Migration Steps

### Step 1: Update Dependencies

```bash
# Remove v3 packages
bun remove @swapkit/plugin-* @swapkit/toolbox-* @swapkit/wallet-*

# Install v4 packages
bun add @swapkit/sdk
# or individual packages
bun add @swapkit/core @swapkit/plugins @swapkit/toolboxes @swapkit/wallets
```

### Step 2: Update Imports

Replace all v3 imports with v4 equivalents:

```diff
-import { THORChainPlugin } from "@swapkit/plugin-thorchain";
-import { EVMToolbox } from "@swapkit/toolbox-evm";
-import { ledgerWallet } from "@swapkit/wallet-ledger";

+import { ThorchainPlugin } from "@swapkit/plugins/thorchain";
+import { getEvmToolbox } from "@swapkit/toolboxes/evm";
+import { ledgerWallet } from "@swapkit/wallets/ledger";
```

### Step 3: Update Client Initialization

```diff
-import { SwapKitCore } from "@swapkit/core";
+import { createSwapKit } from "@swapkit/sdk";

-const client = new SwapKitCore({
+const client = createSwapKit({
  // config
});
```

### Step 4: Update Wallet Connections

The wallet connection pattern remains similar but uses the new import paths:

```typescript
// @noErrorValidation
// Connect wallet (same pattern, new imports)
await client.connectEVMWallet(Chain.Ethereum);
await client.connectKeystore(chains, phrase);
```

## New Features in v4

### Enhanced Solana Support

v4 includes improved Solana token balance fetching with Jupiter API integration for real-time token metadata.

### Dynamic Cosmos Fees

Cosmos toolbox now supports dynamic fee estimation when no fee is provided.

## Common Issues

### TypeScript Errors

If you encounter type errors after migration:

1. Ensure all imports use the new paths
2. Update plugin interfaces to match v4 signatures
3. Check that wallet connection methods match the new types

### Missing Functionality

If a v3 feature is missing:

1. Check if it's been moved to a different module
2. Some features may need different implementation approaches in v4
3. Refer to the API documentation for alternatives
